# GeoGebra 命令白名单：2D 几何关系

同类对象只推荐下面这一种标准写法。能用白名单就不要改写成别的等价命令。

## 标准写法

- 切线对象：`tan1=Tangent(P,c,1)`
- 切点对象：`T=Intersect(tan1,c,1)`
- 垂线对象：`perp1=PerpendicularLine(C,l)`
- 垂足对象：`H=Intersect(perp1,l,1)`
- 角平分线：`bis1=AngleBisector(A,B,C)`

## 使用说明

- `Tangent(...)`、`PerpendicularLine(...)`、`AngleBisector(...)` 先创建线对象
- 如果题目要求切点、垂足、交点，或后续还要继续引用这些点，再显式创建点对象
- 题目已经给出点名时，优先使用题目名称；未给名时，点对象优先使用单个大写字母
- 未给名时，切点优先用 `T`，垂足优先用 `H`，一般交点优先用 `X`
- 切点、垂足、交点优先通过几何命令构造，不要手算坐标
- 依赖对象必须先创建，例如点、圆、直线都要先定义

## 分情况规则

### 1. 在已知圆上点作切线

- 如果题目已经给出圆上点 `P`，例如“过点 `P` 作圆 `c` 的切线”，那么切点就是 `P`
- 这种情况下只需要创建切线对象，不要重复再造一个切点

示例：

```json
{ "id": "tan1", "cmd": "tan1=Tangent(P,c,1)" }
```

### 2. 需要显式切点时

- 如果题目要求切点，或者后续还要继续引用切点，再额外创建点对象
- 切点默认通过“切线与圆的交点”构造

示例：

```json
{ "id": "tan1", "cmd": "tan1=Tangent(A,c,1)" }
{ "id": "T", "cmd": "T=Intersect(tan1,c,1)" }
```

### 3. 需要显式垂足时

- 垂线先创建为线对象
- 如果题目要求垂足，或者后续还要继续引用垂足，再额外创建点对象
- 垂足默认通过“垂线与原直线的交点”构造

示例：

```json
{ "id": "perp1", "cmd": "perp1=PerpendicularLine(C,l)" }
{ "id": "H", "cmd": "H=Intersect(perp1,l,1)" }
```

### 4. 角平分线后续需要交点时

- `AngleBisector(...)` 只先创建线对象
- 如果题目后续还要用角平分线与别的对象的交点，再额外创建点对象

示例：

```json
{ "id": "bis1", "cmd": "bis1=AngleBisector(A,B,C)" }
{ "id": "X", "cmd": "X=Intersect(bis1,lineAC,1)" }
```
